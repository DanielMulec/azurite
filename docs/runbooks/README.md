# Runbooks

Runbooks contain repeatable operational procedures for existing Azurite
behavior. They are not architecture decisions or speculative implementation
plans.

Current runbooks:

- `tailscale-phone-access.md`: start the current local app for private phone QA.

Create a new runbook when a workflow has been implemented and someone will need
to operate or verify it repeatedly. The active Sentry slice owns creating a
Sentry debug runbook after its configuration and runtime behavior exist; until
then, the slice proposal remains the implementation authority.
